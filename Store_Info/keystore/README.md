# Keystore (подпись release APK)

## Что здесь лежит

- `release.jks` — Java KeyStore с приватным ключом для подписи APK. **Один и тот же файл во всех 4 проектах** мастерской (`01_*..04_*`). Содержимое идентично — это намеренно: одна личность разработчика, один ключ.
- `keystore.config.json` — параметры (путь, alias, пароли). Читается `html2apk` при `-Release`.

## Расположение

`<project>/Store_Info/keystore/` — стандартный путь для всех проектов мастерской. html2apk при `-Release` ищет здесь автоматически.

## Зачем общий keystore

Все версии **одного** приложения должны быть подписаны **тем же** ключом. Если пересоздать keystore и подписать обновление другим — РуСтор/Google Play считают это другим приложением, и юзеры с прошлой версии не получат апдейт. **Терять keystore нельзя**.

Один общий keystore на все игры просто экономит управление: один backup, один пароль. Каждая игра при этом имеет свой `appId` (`com.terekh.glittersort`, `com.terekh.words`, ...), магазин их не путает.

## Безопасность

`release.jks` + пароли **коммитятся в git вместе с проектом** — это сознательный компромисс для индивидуального разработчика:

✅ **Что это даёт:**
- Сборка release-APK работает «из коробки» в новом окружении (склонировал репо → собрал → подписано).
- Skill `prepare-release-candidate` находит всё сам, не спрашивает пароль на каждой сборке.

⚠️ **Что это значит для безопасности:**
- Любой, кто получит доступ к репо, сможет подписать APK с этим ключом и опубликовать под видом «настоящего» приложения (если магазин не проверит дополнительные signals, а они обычно не проверяют для уже опубликованных приложений).
- **Если репо приватный** (или эти проекты не публикуются как open-source) — риск практически нулевой.
- **Если когда-нибудь** будешь делать репо публичным или передавать проект — **обязательно** ротируй keystore до этого момента.

## Backup

Несмотря на git: **сделай ещё одну копию `release.jks`** на внешний носитель или в зашифрованное облако. Если потеряешь:
- git history (например, force-push с rewriting) — keystore исчезнет из репо;
- репо упадёт — потеряешь возможность обновлять опубликованные приложения **навсегда**.

Backup location рекомендация: `~/Documents/keystores-backup/terekh-release.jks` + USB-flash где-то отдельно.

## Метаданные keystore

```
Alias:        release
Validity:     36500 days (≈100 лет, до ~2126)
Key algorithm: RSA 2048
Subject DN:   CN=Alexandr Terekhovich, OU=Indie, O=terekh, L=Moscow, ST=Moscow, C=RU
```

## Регенерация (если когда-нибудь понадобится)

```powershell
& "C:\Program Files\Android\Android Studio\jbrin\keytool.exe" -genkeypair -v `
  -keystore "release.jks" `
  -alias release -keyalg RSA -keysize 2048 -validity 36500 `
  -dname "CN=Alexandr Terekhovich, OU=Indie, O=terekh, L=Moscow, ST=Moscow, C=RU" `
  -storepass "terekh-release-2026" `
  -keypass  "terekh-release-2026"
```

⚠️ **Не делай этого** если приложения уже опубликованы — потеряешь связь с ними в магазине.
